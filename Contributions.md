When opening a PR, please fill out the Pull Request Template.

Pull Request Guidelines
PRs must target the develop branch, not main.

PRs must pass project tests if applicable.

PRs must include clear, meaningful commit messages.

Ensure that your code follows the project style and is properly commented.

Large or complex PRs should be discussed first via GitHub Issues before submission.

Coding Standards
Follow PEP8 style guidelines where applicable.

Write readable, maintainable, and modular code.

Prefer clear, descriptive variable and function names.

Add or update documentation where necessary.

Include unit tests or examples if adding new functionality.

Reporting Bugs or Requesting Features
If you discover a bug or want to suggest a feature:

Open a GitHub Issue.

Provide as much detail as possible (steps to reproduce, expected behavior, etc).

Label the issue appropriately (bug, enhancement, question, etc.).

Community Conduct
Be respectful and constructive in discussions.

Keep feedback professional and focused on the work.

Personal attacks, trolling, or disrespectful behavior will not be tolerated.
